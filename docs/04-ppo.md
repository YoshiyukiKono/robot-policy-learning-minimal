# 04 PPO — 何を学習しているのか

PPO は Proximal Policy Optimization の略です。

強化学習アルゴリズムの一つで、方策を少しずつ安全に改善することを目的としています。

## 教師あり学習との違い

教師あり学習では、入力と正解ラベルがあります。

```text
image -> cat
text  -> label
```

しかし強化学習では、各状態に対して「正解行動」が与えられるわけではありません。

代わりに、行動の結果として報酬が返ってきます。

```text
observation -> action -> reward
```

つまり、強化学習では「後から見て良かった行動」を推定する必要があります。

## PPO の流れ

PPO の基本的な流れは次の通りです。

```text
1. 現在の方策で環境を動かす
2. 観測・行動・報酬を集める
3. Advantage を計算する
4. 方策を更新する
5. 繰り返す
```

## Advantage とは何か

Advantage は、その行動が平均的な期待よりどれくらい良かったかを表します。

```text
Advantage = 実際の良さ - 期待される良さ
```

ある行動の Advantage が大きければ、その行動を選びやすくします。

Advantage が小さければ、その行動を選びにくくします。

## なぜ PPO は「少しずつ」更新するのか

強化学習では、方策を急に変えすぎると学習が壊れることがあります。

PPO は、古い方策から新しい方策への変化を制限しながら更新します。

これにより、比較的安定して学習できます。
